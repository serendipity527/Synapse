# 第一阶段功能需求说明书

## 系统核心能力需求
1. 项目目标

构建一个最简的 Java 流程编排引擎，验证“显式数据流”和“命名空间隔离”的核心思想。不涉及复杂路由、异步或异常处理。

2. 核心功能需求

2.1 三级状态管理 (State Management)

系统运行时需维护一个上下文对象 (Context)：

Global: 全局变量（如配置），全流程共享。

Flow: 流程变量（如初始请求参数），单次运行共享。

Nodes: 节点输出隔离区。

节点默认输出放置的区域，不同节点的命名空间是只读的

2.2 显式输入映射 (Explicit I/O Mapping)

配置：在 YAML 中定义节点时，必须声明 inputs 和outputs字段。

 input可以来自于全局变量也可以来自于其他节点的命名空间中的变量，然后output必须要指定显式指定可以是输出到自己的命名空间也可以是去输出到共享的全局变量
